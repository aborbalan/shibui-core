# Atoms Documentation

This section contains detailed documentation for individual **Atom** UI components.

Atoms are the smallest, indivisible UI elements and are strictly **Stateless Components** (no internal state).
They receive all data from parent components via attributes/properties and communicate actions by emitting custom events.

## Available Atoms

-   [Avatar](./avatar.md): User profile picture or placeholder representation.
-   [Badge](./badge.md): Small visual indicator for status, notifications, or labels.
-   [Button](./button.md): A fundamental interactive element.
-   [Card](./card.md): Container for grouping related content.
-   [Checkbox](./checkbox.md): Binary selection control for forms.
-   [Close Button](./close-button.md): Button specifically designed for closing dialogs, modals, or dismissing content.
-   [Copy Button](./copy-button.md): Button to copy content to clipboard.
-   [Glass Card](./glass-card.md): Card component with glassmorphism effect for modern UI aesthetics.
-   [Icon](./icon.md): Visual symbol or graphical representation.
-   [Kbd](./kbd.md): Keyboard key representation for documentation.
-   [Label](./label.md): Text label for form inputs and UI elements.
-   [Lib Panel](./lib-panel.md): Panel container for organizing content sections.
-   [Select Option](./select-option.md): Individual option within a select dropdown.
-   [Skeleton](./skeleton.md): Loading placeholder that mimics content structure.
-   [Spinner](./spinner.md): Loading indicator for asynchronous operations.
-   [Status Dot](./status-dot.md): Small colored indicator for status representation.
-   [Switch](./switch.md): Toggle control for binary on/off states.
-   [Text List](./text-list.md): Styled list component for displaying text items.
